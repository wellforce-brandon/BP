Top Practices to Adopt First

These FOUNDATIONAL practices apply to every repo. If you adopt nothing else, adopt these.

Credential Deny-List

Block AI from reading SSH keys, AWS creds, Docker configs, and npm tokens. Defense-in-depth against prompt injection credential exfiltration.

Prevents: credential leaks via AI

Session Anti-Patterns

Recognize correction loops, kitchen sink sessions, and infinite exploration. Recovery: /clear and re-prompt. Two corrections max before clearing.

Saves: 30-50% wasted tokens per session

Plan Then Execute

Plan in one session, execute in a fresh one. The planning session's research pollutes context. A clean session with just the plan has maximum room for implementation.

Result: cleaner implementations, fewer context-related errors

Compact at 50%

Don't wait for the context limit. Use /compact proactively at 50%. Create HANDOFF.md for multi-session work so the next session picks up exactly where you left off.

Preserves: decision context across sessions

Hierarchical CLAUDE.md

Root CLAUDE.md for project-wide rules (under 200 lines). Subfolder CLAUDE.md for domain-specific rules. .claude/rules/*.md for path-scoped conditional instructions.

Effect: Claude only loads what's relevant to the current task

Read-Only First (RULE 0)

Gather information before taking action. Read-only commands first. Modifications only with user approval. Three safety tiers: always safe, requires approval, never without request.

Prevents: accidental modifications in production

AI Workflow

How to work effectively with AI coding assistants.

Plan in One Session, Execute in Another

Foundational
Planning pollutes context with research and discarded options. By the time you implement, 50%+ of context is spent on exploration. A fresh session with just the plan has maximum context for implementation.

AI Session Anti-Patterns and Recovery

Foundational
Correction Loop: correcting AI 2+ times fills context with failures. Kitchen Sink: mixing unrelated tasks. Infinite Exploration: unscoped "investigate" prompts. Recovery for all three: /clear and re-prompt with better instructions.

Structured Prompting

Recommended
Give specific instructions (file paths, exact commands), not generic advice. AI already knows "write clean code." Tell it things it doesn't know: your naming conventions, import rules, tool commands. If Claude ignores your rules, the CLAUDE.md is too long, not too short.

TDD with AI Assistants

Recommended
Write tests BEFORE asking AI to implement. Tests act as a constraint -- the AI knows exactly what "done" looks like. Without tests, AI tends to over-engineer or add features you didn't ask for.

Parallel Sessions and Fan-Out

Recommended
Writer/Reviewer: implement in one session, review in a fresh one (no bias). Fan-Out: claude -p in a loop for batch migrations. Git Worktrees: parallel agent work without checkout conflicts.

Context Management

Maximize the value of every token in the context window.

Proactive Compaction and Session Handoff

Foundational
Use /compact at ~50% context, not when forced. For multi-session work, create a HANDOFF.md capturing: current state, decisions made, files changed, and next steps. The next session loads only this file.

Subagent Usage Patterns

Recommended
Offload research to subagents -- they protect the main context. Always include "why" in prompts. Spin up parallel Explore subagents when comparing approaches. A subagent that reads 10 files returns only the answer, saving 90% of context.

Shared Agent Memory

Recommended
Version-controlled .claude/agent-memory/ with three files: patterns.md (confirmed codebase patterns), decisions.md (architectural choices with rationale), debugging.md (solutions to recurring problems). Persists across sessions.

Safety

Prevent damage before it happens. Catch problems after they happen.

Credential Path Deny-List

Foundational
Block AI from reading ~/.ssh/, ~/.aws/, ~/.docker/config.json, ~/.npmrc, ~/.git-credentials via deny rules in settings.json. Defense-in-depth against prompt injection credential exfiltration. Source: Trail of Bits.

Read-Only First (RULE 0)

Foundational
Three safety tiers: Always safe (Get-*, Test-*, read commands). Requires approval (Set-*, Remove-*, state changes). Never without explicit request (destructive operations, credential resets).

AI-Assisted Code Review

Recommended
Dedicated reviewer agent with a structured checklist. Findings categorized as CRITICAL (must fix), WARNING (should fix), SUGGESTION (nice to have). Format: [SEVERITY] file:line -- description.

Permissions Pass Gate (RULE 2)

Recommended
For M365/Graph API: privileged operations require cryptographic verification. STOP, ASK for the pass, VERIFY against PFX, then PROCEED. Enforced by PreToolUse hook -- cannot be bypassed.

Claude Configuration

Structure your CLAUDE.md, rules, hooks, and skills for maximum effectiveness.

Hierarchical CLAUDE.md

Foundational
Root CLAUDE.md (under 200 lines) for project-wide rules. Subfolder CLAUDE.md for domain-specific rules. .claude/rules/*.md with path globs for conditional loading. Claude only reads what's relevant.

LL-G Integration (RULE 1)

Foundational
Before writing code, fetch the LL-G index and load HIGH-severity entries for your tech stack. This prevents repeating known failure patterns. Present in every repo's CLAUDE.md.

Path-Scoped Rules

Recommended
Rules in .claude/rules/*.md with glob frontmatter. Only load when editing matching files. Backend rules don't waste context during frontend work.

Hook Configuration

Recommended
PostToolUse hooks for auto-formatting on write. PreToolUse hooks for safety gates. Stop/Notification hooks for audible alerts. Mechanical enforcement, not just instructions.

Skill Frontmatter

Recommended
Right-size each skill with model: haiku|sonnet|opus. Use context: fork for isolation. Haiku for data collection, Sonnet for analysis, Opus for planning.

llms.txt for Every Repo

Recommended
Machine-readable project index at the repo root. What the project is, how it's organized, where key files are. Saves LLMs from reading dozens of files to understand the project.

Documentation

Capture knowledge that outlives a single session.

Plans with Lessons Learned

Foundational
Every plan ends with a Lessons Learned section. Discoveries route to LL-G (gotchas) or BP (best practices), not local files. "Lessons stored locally stay local. Lessons in LL-G benefit every repo."

Architectural Decision Records

Recommended
Document why decisions were made, not just what was chosen. Context, decision, alternatives, consequences. Prevents re-evaluating settled decisions months later.

Reference Files in .claude/references/

Recommended
Stable reference docs (design guardrails, tools, infrastructure) in .claude/references/. Linked from CLAUDE.md with "Read Before" context. Keeps CLAUDE.md lean.

Deployment

Ship with confidence.

Docker Multi-Stage Builds

Recommended
Builder stage compiles, production stage copies only the output. Non-root user, dumb-init for signal handling, npm ci --only=production. Image size drops 2-5x.

Cloudflare Workers with OpenNext

Recommended
Deploy Next.js to Cloudflare Workers via @opennextjs/cloudflare. Zero cold starts, global edge distribution. Build without --turbo (OpenNext requires standard compiler).

Doppler for Secrets

Recommended
Single source of truth for secrets across dev/stg/prd. doppler run for local dev, sync to Cloudflare/Northflank for production. No more scattered .env files.

Linting & Formatting

Consistent code without thinking about it.

Biome Replacing ESLint

Recommended
One Rust-based tool replaces both ESLint and Prettier. 10-100x faster. Linting, formatting, and import sorting in a single pass. No more ESLint/Prettier config conflicts.

Auto-Format on Write Hook

Recommended
PostToolUse hook runs biome check --write after every Edit/Write. Claude's output is automatically formatted to match project conventions. Zero manual formatting passes.

Versioning

Track every change, enforce documentation.

Major.Minor.Patch.Build

Recommended
Four-segment versioning where Build increments on every commit. Trivially correlate deployed versions to exact commits. Never bump Major autonomously -- that's a human decision.

Changelog Enforcement

Recommended
Path-scoped rule fires before git commit, requiring CHANGELOG.md updates. Keep a Changelog format (Added, Changed, Fixed, Removed, Security). Written while context is fresh.

Monorepo

Scale with structure.

pnpm + Turborepo

Recommended
pnpm workspaces for strict dependency resolution, Turborepo for cached task orchestration. packages/, apps/, workers/ with workspace:* references and topological build ordering.

Shared Packages with Layer Dependencies

Recommended
Common @namespace/ prefix, barrel exports from src/index.ts, numbered layer hierarchy (0: utils -> 1: database -> 2: auth -> 3: ui -> 4: apps). Workers never import from apps.

Testing

Confidence through verification.

Vitest for Monorepos

Recommended
Per-package vitest.config.ts with Turborepo orchestration. Native TypeScript support, hot module reloading in watch mode, significantly faster than Jest for TS projects.

Playwright E2E

Optional
Cross-browser E2E testing with auto-start dev server. webServer config for CI, trace: 'on-first-retry' for debugging flaky tests without overhead on passing ones.

Design Systems

Constrain AI-generated UI for consistency.

Guardrail-Driven Design

Recommended
Comprehensive constraints doc in .claude/references/design-guardrails.md. Component size limits, spacing scales, accessibility requirements, performance budgets. Claude reads it before any UI work.

OKLCH Token Architecture

Optional
Perceptually uniform color tokens. Store as raw triplets (0.65 0.2 260), compose at usage (oklch(var(--primary))). Auto-derive 55 semantic roles from 9 seed colors with WCAG AA contrast guarantees.

Environment

Doppler + .env.local Fallback

Recommended
Doppler as primary secrets source, .env.local for offline fallback. Document all env vars in CLAUDE.md with type (secret/public) and purpose so Claude knows what's available.

Error Handling

Structured Logging with Winston

Recommended
JSON logging with timestamps, error stacks, default metadata (service, PID, hostname). Pretty for dev, JSON for prod. Child loggers per component. Console interception for third-party output.

Knowledge Bases

Two-Level Index Pattern

Foundational
Master llms.txt links to per-category indexes. LLMs load the master, identify relevant categories, load only those sub-indexes, then load specific entries. Context-efficient navigation for any knowledge base.
